linux开发：文件的编译流程

post @ 2019-03-27

基本概念

计算机能够执行的是机器语言，高级语言必须翻译为机器语言才能被计算机执行。
这个翻译的方式有两种：编译和解释
1.编译性语言
程序在执行前需要专门的编译过程，生成一个二进制文件。运行时不需要重新编译，效率比较高，比如C。
2.解释性语言
程序在运行时才翻译为机器语言，执行第n条语句之后，再执行第n+1条，每次执行要有一个翻译过程，效率比较低，比如basic、shell等

编译流程

在以前win、51单片机、32、arduino等等上开发的编译单击几个按钮就好。直到遇到了esp8266的ide开发。。。。。。
现在学习linux，必须要弄明白编译流程，才不枉在ee的路上朝cs瞥的这一眼

认识文件扩展名

.c    c原始文件
.h    头文件（预处理文件）
.i    已经预处理过的c原始程序
.s    汇编语言原始程序
.o    目标文件
.a/.so    编译后的库文件

1.编辑阶段

2.预处理阶段

将预处理文件包含进来
注意防止头文件被重复引用如果没有预先设计，那么头文件在预处理阶段会被多次引用，则后面的编译就会出错。
防止头文件被重复引用的方法比较简单，如下

# ifndef HEAD_H_
# define HEAD_H_
    ...
# endif

3.编译阶段

检查代码的规范性、是否有语法错误，以确定代码实际要做的工作。检查无误后，将之翻译成汇编语言
linux命令种要加入选项- g加入调试信息，定位错误

4.汇编阶段

该阶段的工作是将汇编语言汇编成目标文件。该目标文件已经是二进制文件，但还不能执行，缺少的是系统启动代码和引用的库函数

5.链接阶段

链接函数库，需要了解加载器和链接器的基本原理
在Linux下，函数库分为静态库和动态库两种，对应的链接方式为静态链接和动态链接。静态链接时需要把用到库函数的代码全部加到可执行文件中，因此生成文件比较大，但是在运行时就不需要库文件了。动态链接并不会将库函数的代码加入到可执行文件中，而是程序执行时由运行时链接文件加载库，从而节省系统的开销。
GCC在编译时默认使用的是动态库。
嵌入式系统对空间比较敏感，所以在增加应用程序时要综合考虑。如果静态链接后的可执行文件大于动态链接生成的可执行文件和依赖的动态库之和，那么采用动态链接；反之，采用静态链接。
总之要追求系统尺寸最小。

6.调试阶段

得到可执行文件就可以运行了，如果运行出现问题就用调试器来进行调试。

嵌入式及linux系统概述

post @ 2019-03-07

嵌入式系统（Enbedded system）

嵌入式系统的定义

目前国内一个普遍被认同的定义是：以应用为中心、以计算机技术为基础，软、硬件可裁剪，适应应用系统对功能、可靠性、成本、体积、功耗等严格要求的专用计算机系统。

嵌入式系统的特点

1.很强的专用性 ;
2.可裁剪性；
3.系统精简；
4.系统内核小；
5.高实时性的操作系统软件是嵌入软件的基本要求；
6.是将先进的计算机技术、半导体技术和电子技术以及各个行业的具体应用相结合后的产物；
7.嵌入式软件开发要想走向标准化，就必须使用多任务的操作系统；
8.嵌入式系统开发需要专门的开发工具和环境

后PC时代的概念

计算机、通信和消费产品的技术结合起来，以3C产品的形式通过Internet进入家庭，这将是一个庞大的嵌入式应用市场

LINUX系统

为什么嵌入式系统设计这门课要学linux系统？
这门课软件部分学习Linux，硬件部分用一个豪华开发板，开发板上有arduino、arm三四种不同的mcu，以及数不清的外设。也不知道得多贵。软件学linux主要因为系统开源，比较简单。符合嵌入式系统软件部分的可剪裁性和系统精简性。

嵌入式系统的发展历史

1.无操作系统阶段
一般没有操作系统的支持，通过汇编语言对系统进行直接控制
2.简单操作系统阶段
出现了大量具有高可靠性、低功耗的嵌入式CPU
3.嵌入式操作系统阶段
嵌入式操作系统能运行于不同类型的微处理器
4.面向Internet阶段
嵌入式设备与Internet的结合将代表嵌入式系统的未来

linux定义

Linux系统是一个免费使用的类似UNIX操作系统，最初运行在x86体系结构，目前已经被移植到数十种处理器上。Linux系统的开发和维护是由分布在全球各地的数百名程序员完成的，这得益于它的源代码开放特性。Linux不仅支持丰富的硬件设备、文件系统，更主要的是它提供了完整的源代码和开发工具。对于嵌入式开发来说，使用Linux系统可以帮助用户从底层了解嵌入式开发的全过程，以及一个操作系统内部是如何运作的。学习Linux系统开发对初学者有很大的帮助。

linux开发环境

1.独立的Linux系统：需要单独安装在PC机的磁盘分区
2.运行在Windows上的Linux系统：cygwin
3.虚拟机中Ubuntu系统的安装（ubuntu是linux的一种版本，除了它还有red hat等等，版本超过250种）
嵌入式linux开发环境由宿主机（Host，又称为上位机）、目标平台（Target，目标机，又称下位机）、通过网线、串口线、USB线等通信线缆连接而成

参考书目：1.《嵌入式Linux开发详解》刘庆敏 张小亮：初学者友好型，可读性好，比较具体全面

2.《嵌入式linux实践教程》罗怡桂 适合选修完这门课的人看，不是很详细，没有特定的硬件平台，硬件部分可以参考
3.《嵌入式linux c语言应用程序设计与实践》熊茂华 偏软件
4.《ARM嵌入式Linux系统设计与开发》俞辉 偏向具体开发适合最后看，信息量很大，拓宽很好，有具体实例

测量的不确定度

post @ 2018-10-26

测量的不确定度的概念

测量的不确定度

  • 不确定度U：由于测量误差存在而对被测量值不能确定的程度。表明该结果的可信赖程度。
  • 测量的不确定度分为A、B两类
  • A类不确定度UA用对观测进行统计的方法分析评定
  • A类评定必须有重复条件下的测量列
  • B类不确定度UB用非统计方法分析评定
  • 不确定度的合成原则U=√(UA^2+UB^2 )表示由于测量误差的存在而对被测量值不能确定的程度，所有的A类评定和B类评定分量都需要统一到相同的置信区间

    测量量的表示

  • y=y ̅±U
  • 测量不确定度是说明测量结果的参数。在测量结果的完整表示中，应有被测量值的估计值y ̅（最佳值）和测量不确定度U
  • ±U表示被测量值的可能分散区间

    测量的A类不确定度

    测量的A类标准不确定度ΔA

  • 标准不确定度：用标准偏差表示时的测量不确定度。
  • t分布时，Δ_A=Sx=t_0.683 σ_x ̅ =t_0.683√((∑_(i=1)n(x_i-x ̅ )^2 )/n(n-1) )
  • 用测量列算术平均值作为最佳测量值
  • 真值落在区间[x ̅ -Δ_A，x ̅ +Δ_A]的概率为68.3%

    测量的A类拓展不确定度UA

  • 拓展不确定度：用标准不确定度的倍数表示。ua=kp*Δ_A
  • kp分别取2或3时，对应95.5%或99.7%的置信概率
    +大学物理实验约定kp=2

    测量的B类不确定度

    B类不确定度

  • Ub是由不同于观测列统计分析所确定的不确定度，又称非统计不确定度。

    仪器的示值误差和最大允许误差

  • 示值误差：测量仪器示值减去被测量真值之差。最大示值误差也称仪器误差限。
  • 最大允许误差（允差）：国家技术规范、规程等技术文件对给定计量器具在标准条件下极限误差的允许值
  • 预先提出计量要求
  • 判定测量设备合格的依据
  • 一般取仪器的最大（极限）误差作仪器误差限Δ仪
  • Δ仪=量程*级别/100
  • 由仪器说明书或由仪器准确度级别、量程确定
  • 否则可由仪器分度值来判定
  • 对任何测量仪器和仪表，其准确度或仪器误差限应与分度值保持同一数位级
  • 仪器标准差与仪器极限误差关系：S仪=Δ仪/C
  • 正态分布c=3 p=0.683
  • 均匀分布c=√3 p=0.58
  • 三角分布c=√6 p=0.65
  • 大学物理实验合理取u仪=Δ仪（p>=0.95）

    测量者估算产生的误差

  • 估算误差限Δ估：由使用者读数不准或估读引起的。
  • 通常取仪器分度值的0.1至0.5倍，甚至更大。
  • u估=Δ估（p>=0.95）

    测量的B类不确定度

  • B类分量的误差限为ΔB=√(Δ仪^2+Δ估^2）
  • 结果报道要求有较高的置信概率（p>=0.95）

    不确定度与误差的区别

    • 误差：理想概念/科学描述、不易分类与计算
    • 不确定度：定量评价、便于评定与计算

GPIO的概念

post @ 2018-10-01

发现自己也真的是眼高手低了，GPIO的概念都不清楚。。。要反思。。

名称：

GPIO的英文全称General-Purpose Input /Output Ports,中文意思是通用I/O端口。

需求来源

在嵌入式系统中，经常需要控制许多结构简单的外部设备或者电路，这些设备有的需要通过CPU控制，有的需要CPU提供输入信号。并且，许多设备或电路只要求有开/关两种状体就够了，比如LED的亮与灭。

对这些设备的控制，使用传统的串口或者并口就显得比较复杂，所以，在嵌入式微处理器上通常提供了一种“通用可编程I/O端口”，也就是GPIO。

GPIO寄存器

一个GPIO端口至少需要两个寄存器，一个做控制用的“通用IO端口控制寄存器”，还有一个是存放数据的“通用I/O端口数据寄存器”。数据寄存器的每一位是和GPIO的硬件引脚对应的，而数据的传递方向是通过控制寄存器设置的，通过控制寄存器可以设置每一位引脚的数据流向。
（GPIO相关的寄存器有，IOPIN、IOSET、IOCLR、IODIR四个寄存器。)

stm32的引脚复用与重映射

post @ 2018-10-01

端口复用

端口复用概念

STM32有很多内置外设，这些外设的外部引脚都是与GPIO复用的。也就是说，一个GPIO如果可以复用为内置外设的功能引脚，那么当这个GPIO作为内置外设使用的时候，就叫做复用。
例如串口1的发送接收引脚是PA9，PA10，当我们把PA9，PA10不当做GPIO，而当做复用功能串口1的发送接收引脚的时候，就叫做端口复用（引脚复用）。

端口复用硬件原理

STM32系列微控制器IO引脚通过一个复用器连接到内置外设或者模块。
该复用器一次只允许一个外设的复用功能（AF）连接到对应的IO口。这样可以确保公用一个IO引脚的外设之间不会发生冲突。
每个IO引脚都有一个复用器，该复用器采用16路复用功能输入（AF0-AF15），可通过GPIOx_AFRL(针对引脚0-7)和GPIOx_AFRH(针对引脚8-15)寄存器对这些输入进行配置，每四位控制一路复用。

复用端口初始化步骤

1.GPIO 端口时钟使能。要使用到端口复用，当然要使能端口的时钟了。
2.复用的外设时钟使能。比如你要将端口 PA9,PA10 复用为串口，所以要使能串口时钟。
3.端口模式配置。
以查看手册《STM32 中文参考手册 V10》 P110 的表格“8.1.11 外设的 GPIO 配置”（推挽复用输出、浮空输入或者带上拉输入等）

端口重映射

端口重映射概念

为了使不同器件封装的外设 IO 功能数量达到最优，可以把一些复用功能重新映射到其他一 些引脚上。
STM32 中有很多内置外设的输入输出引脚都具有重映射(remap)的功能。我们知道每 个内置外设都有若干个输入输出引脚，一般这些引脚的输出端口都是固定不变的，为了让设计 工程师可以更好地安排引脚的走向和功能，在 STM32 中引入了外设引脚重映射的概念，即一个 外设的引脚除了具有默认的端口外，还可以通过设置重映射寄存器的方式，把这个外设的引脚 映射到其它的端口。

重映射的查询

具体对应关 系《STM32 中文参考手册 V10》的 P116 页“8.3 复用功能和调试配置”有讲解

AFIO时钟

什么时候要开启AFIO呢

操作AFIO的三类寄存器时需要开启：
1.事件控制寄存器(AFIO_EVCR)
2.复用重映射和调试I/O配置寄存器(AFIO_MAPR)
3.外部中断配置寄存器x(AFIO_EXTICRx)
这三类其实是AFIO的全部寄存器， AFIO_EVCR是事件相关的，AFIO_MAPR是与重映射和调试I/O相关的，AFIO_EXTICRx是与外部中断相关的。因此管脚重映射和调试I/O配置、外部中断、事件时需开启AFIO。

重映射步骤

1.重映射GPIO 端口时钟使能。
2.外设时钟使能。
3.使能 AFIO 时钟
4.开启重映射
